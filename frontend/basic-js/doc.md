## Основы
- Инструкция (statement) - команда для компьютера выполнить что-либо. Это команда в коде, которая выполняет определённое действие, но не обязательно возвращает значение. Инструкции управляют потоком выполнения программы, задают её структуру. Код в JS - набор инструкций, отделённых друг от друга `;`. Инструкция – единица исполнения, построчно запускаемая интерпретатором в строгом порядке, указанным в коде. Инструкция - это конструкция, выполняющая команду. Примеры инструкций: объявление переменной (`let, var, const`), присваивание (`=`), условный блок (`if else`), цикл (`for, while`) и работа с функциями (`function, return`).

Инструкции не являются выражениями, так как не вычисляются в значение. Их нельзя использовать в местах, где ожидается выражение (например, в качестве аргумента функции).

- Синтаксические ошибки (SyntaxError) возникают в результате выполнения программы с грамматическими ошибками
`Missing ('`

- Оператор - символ или ключевое слово в ЯП, обозначающее конкретную операцию. Это синтаксическая конструкция, инструмент для записи операции в коде.
Оператор - знак операции, например, `+`. Операторы выполняют операции над операндами (определёнными значениями). Операторы представлены одним (`+`) или несколькими (`**`) символами, реже - словом (`delete`, `typeof`, `instanceof`, `void`, `in`).

- Операция - действия, выполняемые над данными (операндами). Операция определяет действия: сложить, сравнить, объединить. Это элемент, вычисляющий значение. Операция определяет способ обработки данных.

- Операнд - данные, над которыми проводится операция: переменные, литералы или выражения, которые вычисляются в конкретные значения (`a+b`, `getPrice()`). Вызов функции может быть операндом, если участвует в операции `getPrice() * 2`.

**Пример**
`8 + 2` - операция (сложение)
`8, 2` - операнды
`+` - оператор

- Выражение (expression) - любое действие, вычисляемое в какое-либо значение. Это комбинация значений (операторов и операндов), вычисляемых в единственное значение.

### Разница понятий операция, выражение, инструкция
| Критерий | Операция | Выражение | Инструкция |
|----------|----------|-----------|-----------|
| **Цель** | Выполнить действие над операндами | Вычислить значение | Выполнить действие (управлять потоком выполнения) |
| **Результат** | Изменение состояния операндов или вычисление значения | Конкретное значение | Обычно `void` (нет значения) |
| **Пример** | `+` (сложение), `-` (вычитание) | `5 + 3`, `getPrice()`, `a * b + c` | `if (condition) { ... }`, `for (let i = 0; i < 10; i++) { ... }` |

Так, операции определяют действия, выражения вычисляют значения, а инструкции управляют выполнением программы.
Операция - логика, выражение - результат, инструкция - архитектура.


- Коммутативная операция - при перестановке операндов результат неизменен.

- В JS нет различий между рациональными числами (0.5) и натуральными (10), их можно использовать в одних операциях
`0.2 * 2 = 0.4`.

- Неточность операций с рациональными числами обусловлена ограничением вычислительных мощностей. Объём памяти, в отличие от чисел, конечен (бесконечное количество чисел требует бесконечного количетсва памяти). Натуральные числа решают проблему хранения значений по верхней границе (есть максимальное число Number.MAX_SAFE_INTEGER), то с рациональными числами это не работает, так как они не представлены в виде цепочек: между 0.1 и 0.2 большая пропасть значений, однако для хранения в памяти рациональных чисел есть специальный стандарт.

### BigInt
BigInt добавлен в ES2020 для работы с целыми числами произвольной длины. Он решает проблему потери точности при работе с очень большими целыми числами, которые выходят за пределы безопасного диапазона для Number.

BigInt использует для гарантии точности при работе с большими числами.

**Пример неточности**
Number.MAX_SAFE_INTEGER +1 (+2) дадут одинаковый результат
```javascript
Number.MAX_SAFE_INTEGER + 1; // 9007199254740992
Number.MAX_SAFE_INTEGER + 2; // 9007199254740992
```

BigInt решает проблему
```javascript
console.log(
    BigInt(Number.MAX_SAFE_INTEGER) + 1n, // 9007199254740992n
    BigInt(Number.MAX_SAFE_INTEGER)+ 2n, // 9007199254740993n
);
```

**Алгоритм**
Число x должно делиться без остатка на любое из диапазона от 2 до x-1, если такого числа нет, то это простое число. На практике проверяем не все числа из диапазона, а до половины числа. Например, 11 не делится на 2, 3, 4, 5. И дальше гарантированно не будет делиться на числа больше своей половины. Значит, можно оптимизировать процесс, проводя деление только до `x/2`.

```javascript
const isPrime = (number) => {
    if (number < 2) return false;

    let divider = 2;

    while (divider <= number / 2) {
        if (number % divider === 0) return false;
        divider++;
    }

    return true;
}

console.log(
    isPrime(1),
    isPrime(2),
    isPrime(7),
    isPrime(11),
    isPrime(23),
)
```
Если во время последовательного деления на числа до `x / 2` находится хоть одно, которое делит без остатка, то переданный аргумент - не простое число, а значит дальнейшие вычисления не имеют смысла. В этом месте стоит возврат false. Если цикл отработал целиком, можно сделать вывод, что число - простое, так как не было ни одного числа, которое делит число без остатка.

const hasChar = (str, char) => {

    }

## Цикл For
Цикл `while` идеален для ситуаций, когда количество итераций неизвестно заранее (поиск простого числа).
Цикл `for` используется при известном заранее количестве итераций.

Переворот строки
```javascript
const reverseString = (str) => {
    let result = '';

    for (let i = 0; i < str.length; i++) result = `${str[i]}${result}`;

    return result;
}

console.log(
    reverseString('Hexslet')
);
```

В определении цикла for в круглых скобках есть три выражения:
- Начальное значение счётчика (код выполняется один раз при первой итерации)
- Предикат (условие повторения циклов). Код выполняется на каждой итерации
- Описание изменения счётчика. Код выполняется в конце каждой итерации

Пример бесконечного цикла
```javascript
let i = 1;
for (;;) {
    console.log(i)l
    i++;
}
```



### Задача: переставить два символа строки местами, если нечетная длина - последний символ оставить без изменений. Пример: move -> omev
Решение учителя
```javascript
const encrypt = (str) => {
  let result = "";
  for (let i = 0; i < str.length; i += 2) {
    const nextSymbol = str[i + 1] || "";
    result += `${nextSymbol}${str[i]}`;
  }
  return result;
};

console.log(
    encrypt('move!')
);
```

Моё решение
```javascript
function encrypt(str, stringBit=2, step=0) {
  let formattedString = "";

  for (let i = 0; i < str.length; i++) {
    formattedString += str.slice(step, stringBit).split("").reverse().join("");
    step += 2;
    stringBit += 2;
    i++;
  }

  return formattedString;
}

console.log(encrypt("attack"));
```

Через match
```javascript
const encrypt = (str) => str.match(/.{1,2}/g).map(pair => pair.length === 2 ? pair[1] + pair[0] : pair).join('');

);
```

Через рекурсию

const encrypt = (str) => {

### Переворот строки в JS

Положительный индекс
```javascript
let name = 'oshust';
let reverseName = '';

for (let i = name.length - 1; i >= 0; i--) reverseName += name[i];

console.log(reverseName);
```

split
```javascript
console.log(
    'tsuhso'.split("").reverse().join("")
); 
```

Рекурсия
```javascript
function reverseString(str) {
    if (str === '') return '';
    return reverseString(str.substr(1)) + str.charAt(0);
    // substr(1) убирает первый символ и передаёт оставшуюся часть в следующий вызов (shust)

    // chatAt(0) берёт отрезанный символ и добавляет его в конец.
}

let name = 'oshust';
let reverseName = reverseString(name);
console.log(reverseName);
```
Как работает на другом примере
```javascript
reverseString("abc")
├── reverseString("bc") + "a"
│   ├── reverseString("c") + "b"
│   │   ├── reverseString("") + "c"  ← базовый случай!
│   │   └── возвращает "c"
│   └── возвращает "c" + "b" = "cb"
└── возвращает "cb" + "a" = "cba"
```

reduce
```javascript
let name = 'oshust';
let reverseName = name.split('').reduce((reversed, char) => char + reversed, '');

console.log(reverseName);
```
reduce превращает массив в одно значение. Проходит по массиву символов и на каждом шаге добавляет текущий символ в начало аккумулирующей строки.

spread
```javascript
let name = "oshust";
let reverseName = [...name].reverse().join('');

console.log(reverseName);
```
Более современный синтаксис, spread-operator аналогично преобразует строку в массив символов.

for...of + unshift()
```javascript
let name = 'oshust';
let chars = [];

for (let char of name) {
    chars.unshift(char);
}

let reverseName = chars.join('');
console.log(reverseName);
```

### Вывести символы в верхнем регистре
Решение по курсу
```javascript
const makeItFunny = (string, n) => {
    let i = 0;
    let result = '';

    while (i < string.length) {
        const current = string[i];
        if ((i+1) % n === 0) {
            result = `${result}${current.toUpperCase()}`;
        } else {
            result = `${result}${current}`;
        }
        i++;
    }

    return result;
}

console.log(
    makeItFunny('I never look back', 3)
);
```

split + map + join
```javascript
const makeItFunny = (str, n) => str.split('').map((char, i) => (i + 1) % n === 0 ? char.toUpperCase() : char).join('');
```

replace + регулярные выражения
```javascript
const makeItFunny = (str, n) => str.replace(/./g, (char, i) => (i + 1) % n === 0 ? char.toUpperCase() : char);

console.log(makeItFunny('I never look back', 3));
```

spread-оператор (корректно работает с Unicode и эмодзи)
```javascript
const makeItFunny = (str, n) => [...str].map((c, i) => (i + 1) % n ? c : c.toUpperCase()).join('');

console.log(makeItFunny('I never look back', 3));
```
[...str] предпочтительнее split, если в строке могут быть суррогатные пары (например, эмодзи)
